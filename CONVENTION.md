# 기술 스택 및 컨벤션

## 프론트엔드
- **React**: UI 라이브러리
- **TypeScript**: 정적 타입 지원
- **React Router**: SPA 라우팅
- **Axios**: API 통신
- **Jest**: 컴포넌트 테스트

## 백엔드
- **Node.js**: 런타임 환경
- **Express**: 웹 프레임워크
- **TypeScript**: 정적 타입 지원
- **JWT**: 인증 및 인가
- **MySQL**: 관계형 데이터베이스
- **Sequelize**: ORM(Object-Relational Mapping)
- **Jest**: 테스트 프레임워크

## 코드 컨벤션

### 공통
- 타입스크립트 사용, 타입 명시 필수
- 함수 및 변수명은 카멜케이스(camelCase) 사용
- 클래스명은 파스칼케이스(PascalCase) 사용
- 주석은 한글 또는 영어로 명확하게 작성
- 불필요한 코드 및 주석은 커밋 전 제거

### 프론트엔드
- 컴포넌트 파일명은 파스칼케이스 사용 (예: `UserProfile.tsx`)
- 스타일은 CSS-in-JS 또는 모듈화된 CSS 사용
- API 통신은 Axios로 통일, 에러 핸들링 필수
- 라우팅은 React Router로 관리
- 테스트는 Jest 및 React Testing Library 권장

### 백엔드
- 라우터, 컨트롤러, 서비스, 모델 분리
- API 응답은 일관된 JSON 포맷 사용
- 인증/인가는 JWT로 처리
- DB 접근은 Sequelize ORM 사용
- 테스트는 Jest로 작성

## 커밋 컨벤션

- 커밋 메시지는 반드시 영어 소문자 태그로 시작 (예: `feat:`, `fix:` 등)
- 태그와 본문 사이에는 콜론(:)과 한 칸의 공백을 둠
- 제목(Subject)은 50자 이내로 간결하게 작성
- 본문(Body)은 선택 사항이며, 필요 시 한 줄 띄워 상세 설명 작성 (72자 내외로 줄바꿈)
- 이슈 트래킹 시, 본문 또는 푸터에 `Refs: #이슈번호` 또는 `Fixes: #이슈번호` 명시
- 커밋 예시:
  ```
  feat: 사용자 로그인 기능 추가

  JWT 기반 인증 로직 구현 및 테스트 코드 작성
  Refs: #12
  ```
- 커밋 태그 종류:
  - feat: 새로운 기능 추가
  - fix: 버그 수정
  - docs: 문서 수정
  - style: 코드 포맷팅, 세미콜론 누락 등 비즈니스 로직에 변화 없는 경우
  - refactor: 코드 리팩토링
  - test: 테스트 코드 추가/수정
  - chore: 빌드 업무 수정, 패키지 매니저 수정 등
  - perf: 성능 개선
  - ci: CI 관련 설정/스크립트 변경
  - revert: 커밋 되돌리기
